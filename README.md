# 동아리방 보드게임 대여와 약속 관리 서비스

동아리방의 보드게임 대여, 반납, 게임 약속, 관리자 운영을 한 곳에서 처리할 수 있도록 만든 웹 서비스입니다. 수기로 관리하던 보드게임 목록과 대여 현황을 웹 기반으로 전환해 중복 대여를 막고, 운영자가 회원/게임/대여/로그를 빠르게 확인할 수 있도록 구성했습니다.

- 배포 도메인: `https://giusite.site`
- 개발 기간: 2026.04.28 - 현재 진행 중
- 프로젝트 유형: 동아리 운영 자동화 웹 서비스
- 주요 사용자: 동아리 회원, 동아리방 관리자

## 문제 정의

- 동아리방 보드게임 수가 많아지면서 대여 가능 여부를 수기로 확인하기 어려웠습니다.
- 누가 어떤 게임을 언제 빌렸는지, 반납 예정일이 언제인지 운영자가 한눈에 보기 어려웠습니다.
- 회원, 게임 목록, 대여 요청, 약속 현황, 운영 로그가 흩어져 있어 관리 비용이 높았습니다.
- 기존 보드게임 목록은 엑셀로 관리되고 있어 웹 서비스에서도 엑셀 기반 업로드/백업 흐름이 필요했습니다.

## 해결 방향

- 회원용 화면에서는 보드게임 검색, 필터, 대여 요청, 약속 생성을 빠르게 수행할 수 있도록 구성했습니다.
- 관리자 화면에서는 승인 대기, 운영 로그, 대여 관리, 회원 관리, 약속 관리, 게임 수정/등록, 엑셀 업로드를 탭 형태로 분리했습니다.
- 대여 중인 게임은 상태값으로 관리하고, 대여/반납 이력과 사진을 함께 저장해 운영자가 실제 반납 상태를 확인할 수 있도록 했습니다.
- 게임 약속 기능을 추가해 회원들이 테이블, 시간, 인원, 게임을 선택해 모임을 만들고 참여할 수 있도록 했습니다.

## 주요 화면과 기능

### 회원용 보드게임 대여 화면

- 전체 보드게임 수, 대여 가능 수, 승인 대기 수, 예정 약속 수를 요약 카드로 표시
- 게임명, 상태, 인원, 베스트 인원, 시간, 장르, 웨이트 기준 검색/필터 제공
- 대여 가능한 게임만 바로 대여 요청할 수 있도록 상태와 액션을 분리
- 많은 보드게임 데이터를 다루기 위해 페이지네이션 적용

### 관리자 데이터 관리

- 회원 수, 보드게임 수, 예정 약속, 운영 로그, 승인 대기 현황을 대시보드에서 요약
- 대여/반납 승인 요청을 관리자 화면에서 처리
- 대여 중인 게임, 대여자, 대여 시각, 반납 예정일, 대여 사진을 함께 확인
- 운영 로그와 게임 약속 로그를 별도 영역에서 확인해 운영 이력을 추적

### 게임 약속 만들기

- 제목, 테이블, 시작 시간, 최대 인원, 설명을 입력해 게임 약속 생성
- 약속에 사용할 보드게임을 검색 후 선택
- 테이블별 수용 인원을 반영해 모임 규모를 제한
- 생성/완료/취소된 약속은 로그로 남겨 관리자 화면에서 추적

### 게임 데이터 관리

- 관리자 화면에서 게임명, 인원, 베스트 인원, 시간, 수량, 장르, 존재 여부, 웨이트, 정보 사이트, 비고 수정
- 신규 게임 등록 기능 제공
- 엑셀 업로드로 전체 보드게임 DB 교체
- 엑셀 다운로드로 현재 게임 DB 백업
- 대여 중인 게임이 있을 때는 DB 교체를 막아 운영 데이터가 꼬이지 않도록 처리

## 기술 구현 포인트

- Next.js App Router 기반으로 사용자 화면과 관리자 화면을 구성했습니다.
- Prisma와 PostgreSQL로 회원, 세션, 보드게임, 대여, 대여 요청, 대여 사진, 약속, 참여자, 알림 로그 모델을 설계했습니다.
- 대여 처리는 트랜잭션으로 묶어 동시에 같은 게임을 대여하는 상황을 방지했습니다.
- 세션은 DB에 저장하고 만료 시간을 두어 로그인 상태를 관리했습니다.
- 로그인, 회원가입, 대여, 반납, 약속 생성, 참여 요청에 rate limit을 적용했습니다.
- ExcelJS를 사용해 보드게임 목록 업로드/다운로드 기능을 구현했습니다.
- Discord Webhook과 Railway Cron을 이용해 반납 요청 및 반납 지연 알림을 보낼 수 있도록 구성했습니다.

## 로컬 실행 방법

프로젝트 변경 기준은 [프로젝트 가이드라인](docs/project_guidelines_ko.md)을 함께 참고합니다.

1. 의존성 설치

```bash
npm install
```

2. 환경변수 설정

`.env.example`을 참고해 `.env`를 생성합니다.

```env
DATABASE_URL="postgresql://USER:PASSWORD@HOST:PORT/DATABASE?schema=public"
SESSION_SECRET="change-this-to-a-long-random-secret"
DISCORD_WEBHOOK_URL="https://discord.com/api/webhooks/..."
```

3. DB 마이그레이션

```bash
npm run db:migrate
```

4. 샘플 데이터 입력

```bash
npm run db:seed
```

기본 테스트 계정은 아래와 같습니다.

```text
관리자 ID: admin
비밀번호: admin1234

일반 사용자 ID: user
비밀번호: user1234
```

5. 개발 서버 실행

```bash
npm run dev
```

브라우저에서 `http://localhost:3000`으로 접속합니다.

## 주요 스크립트

```bash
npm run dev              # 개발 서버 실행
npm run build            # Prisma generate + Next build + standalone static 준비

npm run db:migrate       # 로컬 DB 마이그레이션

npm run db:import-games  # 게임 데이터 import
npm run notify:loans     # 반납 지연 알림 실행
```

## Railway 배포

1. Railway에서 PostgreSQL 플러그인을 추가합니다.
2. 서비스 환경변수에 `DATABASE_URL`, `SESSION_SECRET`, `DISCORD_WEBHOOK_URL`을 설정합니다.
3. 배포 후 아래 명령으로 DB를 준비합니다.

```bash
npm run db:deploy
npm run db:seed
```

4. Railway 배포 로그에서 `prisma generate`와 `next build` 성공 여부를 확인합니다.
5. 배포 URL 또는 `https://giusite.site`로 접속해 관리자 계정으로 로그인합니다.

## Discord 알림과 Cron

반납 요청 알림과 반납 지연 알림은 Discord Webhook으로 발송합니다.

- 반납 요청 알림은 사용자가 반납 요청을 올릴 때 즉시 발송됩니다.
- 반납 지연 알림은 Railway Cron 서비스에서 `npm run notify:loans`를 실행하도록 설정합니다.
- Railway Cron은 UTC 기준입니다. 매일 오전 10시 KST에 실행하려면 `0 1 * * *`로 설정합니다.
- Cron 전용 서비스를 같은 저장소로 만들 때는 Config file path를 `/railway.notify-loans.json`로 지정합니다.

## 운영 로그 확장 기준

운영자가 추적해야 하는 사용자/관리자 행동은 `src/lib/activity-log.ts`의 `createGeneralActivityLog`로 남깁니다.

- 기존 대여/반납과 약속 로그는 전용 로그 모델을 유지합니다.
- 회원가입, 비밀번호 변경/초기화, 계정 삭제, 보드게임 생성/수정/import, 공지 생성/수정/삭제처럼 운영 이력으로 확인해야 하는 행동은 범용 관리 작업 로그에 기록합니다.
- 새 기능을 추가할 때는 액션 성공 직후 `category`, `action`, `actor`, `target`, `message`, 필요한 `metadata`를 함께 저장합니다.
- 관리자 화면 `/admin/logs`와 CSV export는 범용 관리 작업 로그를 함께 보여줍니다.

## 보드게임 엑셀 양식

관리자 업로드는 첫 번째 시트의 A-K 컬럼을 읽습니다.

```text
제목 | 인원(명) | 베스트 인원 | 시간(분) | 수량(개) | 비고 | 소유자 | 장르 | 존재 여부 | 난이도(웨이트) | 보드게임 정보 사이트
```

- `소유자` 열은 DB에 저장하지 않습니다.
- `보드게임 정보 사이트` 열은 관리자 DB 관리용으로 저장하고 사용자 대여 화면에는 표시하지 않습니다.
- 빈칸은 빈칸으로 저장하고 화면에서도 표시를 강제하지 않습니다.
- `시간(분)`과 `난이도(웨이트)`는 `∞`, `30~60`, `1.28` 같은 원본 표현을 보존합니다.
- 업로드는 현재 게임 목록을 엑셀 기준으로 교체합니다.
- 대여 중인 게임이 있으면 업로드를 막습니다.

## 기능 확인 체크리스트

- 회원가입 후 자동 로그인되는지 확인
- 로그인 실패가 반복될 때 rate limit 메시지가 나오는지 확인
- 일반 사용자가 보드게임을 검색하고 대여 요청할 수 있는지 확인
- 일반 사용자가 반납 요청과 사진 업로드를 할 수 있는지 확인
- 관리자가 대여/반납 요청을 승인 또는 거절할 수 있는지 확인
- 관리자가 게임을 추가/수정할 수 있는지 확인
- 관리자가 회원 권한을 수정하고 비밀번호를 `1981`로 초기화할 수 있는지 확인
- 초기화된 회원이 `/account/password`에서 새 비밀번호를 설정할 수 있는지 확인
- 관리자가 엑셀 파일을 업로드해 게임 DB를 교체할 수 있는지 확인
- 관리자가 `/admin/games/export`에서 현재 게임 DB를 내려받을 수 있는지 확인
- 개최자 또는 관리자가 게임 약속을 완료/취소해서 목록에서 제거할 수 있는지 확인
- 일반 사용자 계정이 `/admin`에 접근할 수 없는지 확인
- 다른 회원이 이미 대여 중인 게임을 다시 대여할 수 없는지 확인
- 모바일 너비에서도 카드와 폼이 겹치지 않는지 확인
